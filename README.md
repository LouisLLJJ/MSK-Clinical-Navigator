# MSK Clinical Navigator MVP

痛みの部位・動き・キーワードから筋骨格系の候補を絞り込み、関連組織、検査、治療候補、セルフケアを確認するローカルWebアプリです。検査結果を選ぶと候補の関連度が再計算されます。

## 重要な位置づけ

このアプリは診断器ではなく、医療従事者や学習者が臨床推論を整理するための試作です。表示する「関連度」は入力項目に対するルールベースの一致度であり、疾患確率、感度・特異度、診断確率ではありません。単一の整形外科的検査だけで診断を確定しないでください。

急な麻痺、進行する筋力低下、膀胱直腸障害、会陰部感覚低下、発熱、重大外傷、胸痛・息切れ、がん既往など、重篤疾患を疑う所見がある場合は、このアプリの検索より適切な緊急評価を優先してください。

## 起動方法

依存パッケージやビルドは不要です。

1. このフォルダでローカルサーバーを起動します。

   ```powershell
   python -m http.server 8000
   ```

2. ブラウザで `http://localhost:8000` を開きます。

`index.html` を直接開いても基本機能は動きますが、JSONの読み書きやブラウザ差を避けるためローカルサーバーを推奨します。

## 使い方

1. 「痛みの部位」「痛い・困る動き」「キーワード」を入力します。
2. 候補が関連度順に並ぶので、確認したい候補を選びます。
3. 関連する筋・関節・神経と検査項目を確認します。
4. 検査結果を「陽性」「陰性」「未実施」から選びます。全候補の関連度が再計算されます。
5. 治療候補、セルフケア、注意・紹介目安を確認します。
6. 右上の「データ編集」から候補を追加・編集・削除できます。

編集内容はブラウザの `localStorage` に保存されます。「JSONを書き出す」でバックアップし、「JSONを読み込む」で別環境へ移せます。

## 現在のスコア仕様

- 部位の完全一致: `+35`
- 動きの完全一致: `+30`
- キーワード一致: 1語ごとに `+18`
- 検査陽性: `+14`
- 検査陰性: `-9`

スコアは最大値を100として画面表示用に正規化します。この値に医学的な確率の意味はありません。今後、妥当性を検証した尤度比やベイズ更新を使う場合は、対象集団、事前確率、検査手順、参照基準を明示してください。

## データ構造

候補データは `app.js` の `seedData` にあります。

```js
{
  id: "一意なID",
  name: "候補名",
  summary: "概要",
  regions: ["部位"],
  movements: ["誘発動作"],
  keywords: ["特徴"],
  muscles: ["筋"],
  joints: ["関節・組織"],
  nerves: ["神経"],
  tests: [{ id: "test-id", name: "検査名", finding: "陽性所見" }],
  treatments: ["治療候補"],
  selfCare: ["セルフケア"],
  cautions: ["注意・紹介目安"]
}
```

## 今後の拡張

1. **データと画面の分離**: `seedData` をJSONまたはデータベースへ移し、版管理・レビュー承認・出典・更新日を候補単位で保持する。
2. **医学的根拠の構造化**: 各検査に対象集団、手順、感度、特異度、尤度比、出典、エビデンス確実性を追加する。
3. **安全性**: 検索前にレッドフラッグと神経学的スクリーニングを実装し、条件に応じて通常の候補表示を中断する。
4. **スコアリング**: 専門家レビューと実データで重みを検証し、候補同士で共有される検査結果も反映する。
5. **症例保存**: IndexedDBやバックエンドを使い、匿名化した症例、経過、アウトカムを保存する。個人情報を扱う場合は認証、暗号化、監査ログ、同意、関連法令への対応が必要。
6. **共同編集**: 管理画面、権限、変更履歴、レビュー・公開ワークフローを追加する。
7. **PWA化**: Service Workerとマニフェストを追加し、スマホのホーム画面追加とオフライン利用に対応する。
