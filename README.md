# MSK Clinical Navigator MVP

痛みの部位、複数の症状・動き、キーワードから筋骨格系の候補を絞り込み、関連組織、検査、治療候補、セルフケアを確認するWebアプリです。臨床推論、症状・部位、解剖、検査、治療のデータが相互に連結され、どのページで編集しても関連する表示と検索が更新されます。

公開版: https://louislljj.github.io/MSK-Clinical-Navigator/

## 重要な位置づけ

このアプリは診断器ではなく、医療従事者や学習者が臨床推論を整理するための試作です。表示する「関連度」は入力項目に対するルールベースの一致度であり、疾患確率、感度・特異度、診断確率ではありません。単一の整形外科的検査だけで診断を確定しないでください。

急な麻痺、進行する筋力低下、膀胱直腸障害、会陰部感覚低下、発熱、重大外傷、胸痛・息切れ、がん既往など、重篤疾患を疑う所見がある場合は、このアプリの検索より適切な緊急評価を優先してください。

## 起動方法

依存パッケージやビルドは不要です。

1. このフォルダでローカルサーバーを起動します。

   ```powershell
   python -m http.server 8000
   ```

2. ブラウザで `http://localhost:8000` を開きます。

`index.html` を直接開いても基本機能は動きますが、JSONの読み書きやブラウザ差を避けるためローカルサーバーを推奨します。

## 使い方

画面上部には次の5ページがあります。

- **臨床推論**: 症状入力、SFMA、候補順位、検査結果、治療候補を表示
- **症状・部位**: 部位、動作・活動、症状・特徴を一覧・編集
- **解剖**: 筋、関節・組織、神経を一覧・編集
- **検査**: 検査名、陽性所見、関連候補を一覧・編集
- **治療**: 治療候補、セルフケア、関連候補を一覧・編集

1. 「痛みの部位」を1つ選びます。必要な部位がなければ入力欄から追加できます。
2. 「症状・痛い動き」は複数選択できます。必要に応じてキーワードも入力します。
3. SFMAを実施した場合は、頸部屈曲・伸展・左右回旋、多分節屈曲・伸展をFN・FP・DN・DPで入力します。
4. SFMAブレイクアウトを実施した場合は、頸椎上部・頸椎下部・胸椎・肩甲胸郭・肩関節・腰椎・股関節・膝・足関節をFN・FP・DN・DPで入力します。
5. 候補が関連度順に並ぶので、確認したい候補を選びます。
6. 関連する筋・関節・神経と検査項目を確認します。
7. 検査結果を「陽性」「陰性」「未実施」から選びます。全候補の関連度が再計算されます。
8. 治療候補、セルフケア、注意・紹介目安を確認します。
9. 右上の「候補編集」から候補そのものを追加・編集・削除できます。

臨床推論画面で入力した部位、症状選択、キーワード、SFMA、SFMAブレイクアウト、検査結果、編集中の追加項目は自動保存されます。ブラウザを閉じても同じ端末・同じブラウザでは復元されます。

臨床推論画面の候補詳細からも直接編集できます。

- 「この候補を編集」: 候補名、概要、部位、症状、解剖、検査、治療、注意事項を編集
- 各タグ横の「編集」: 部位、症状、解剖項目をその場で編集
- 検査名横の「編集」: 検査名、陽性所見、関連候補を編集
- 治療・セルフケア横の「編集」: 介入名、説明、関連候補を編集
- 「臨床画面でデータ追加・編集」: 部位、症状、解剖、検査、治療、セルフケアを入力候補から選び、選択中の候補へ追加

## 連結データの編集

各辞書ページのカードにある「編集」を押すと、名前、分類、説明、関連候補を変更できます。

- 解剖名を変更すると、関連する全候補の解剖表示も変更されます。
- 症状や部位の関連候補を変更すると、検索条件と候補スコアに反映されます。
- 検査の陽性所見を変更すると、関連する全候補の検査表示が変わります。
- 治療やセルフケアを変更すると、関連する全候補の表示が変わります。
- 臨床推論画面の症状・解剖タグを押すと、対応する辞書ページへ移動できます。

辞書ページで削除すると、その項目は関連するすべての候補から外れます。削除前にJSONを書き出してバックアップすることを推奨します。

編集内容はブラウザの `localStorage` に保存されます。「JSONを書き出す」で候補、追加部位、辞書説明を含む全データをバックアップし、「JSONを読み込む」で別環境へ移せます。旧形式の候補配列JSONも読み込めます。

## 現在のスコア仕様

- 部位の完全一致: `+35`
- 選択した症状・動きの一致: 1項目ごとに `+22`
- キーワード一致: 1語ごとに `+18`
- 検査陽性: `+14`
- 検査陰性: `-9`
- 関連するSFMAパターンのFP・DP: 1項目ごとに `+7`
- 関連するSFMAパターンのDN: 1項目ごとに `+3`

スコアは最大値を100として画面表示用に正規化します。この値に医学的な確率の意味はありません。SFMA結果も診断名を示すものではなく、頸部または多分節パターンに関連する候補への補助点としてのみ使います。今後、妥当性を検証した尤度比やベイズ更新を使う場合は、対象集団、事前確率、検査手順、参照基準を明示してください。

## データ構造

候補データは `app.js` の `seedData` にあります。

```js
{
  id: "一意なID",
  name: "候補名",
  summary: "概要",
  regions: ["部位"],
  movements: ["誘発動作"],
  keywords: ["特徴"],
  muscles: ["筋"],
  joints: ["関節・組織"],
  nerves: ["神経"],
  tests: [{ id: "test-id", name: "検査名", finding: "陽性所見" }],
  treatments: ["治療候補"],
  selfCare: ["セルフケア"],
  cautions: ["注意・紹介目安"]
}
```

## 今後の拡張

1. **データと画面の分離**: `seedData` をJSONまたはデータベースへ移し、版管理・レビュー承認・出典・更新日を候補単位で保持する。
2. **医学的根拠の構造化**: 各検査に対象集団、手順、感度、特異度、尤度比、出典、エビデンス確実性を追加する。
3. **安全性**: 検索前にレッドフラッグと神経学的スクリーニングを実装し、条件に応じて通常の候補表示を中断する。
4. **スコアリング**: 専門家レビューと実データで重みを検証し、候補同士で共有される検査結果も反映する。
5. **症例保存**: IndexedDBやバックエンドを使い、匿名化した症例、経過、アウトカムを保存する。個人情報を扱う場合は認証、暗号化、監査ログ、同意、関連法令への対応が必要。
6. **共同編集**: 管理画面、権限、変更履歴、レビュー・公開ワークフローを追加する。
7. **PWA化**: Service Workerとマニフェストを追加し、スマホのホーム画面追加とオフライン利用に対応する。
